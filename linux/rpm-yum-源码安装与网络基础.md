# RPM、YUM、源码安装与网络基础

## 概览

- RPM 是 CentOS/RHEL 系软件包底层管理机制，安装、查询、卸载方便，但不能自动解决依赖。
- YUM 基于 RPM，核心价值是自动处理依赖和仓库管理。
- 源码安装通常用于 yum 版本太低、默认编译参数不满足需求或需要自定义模块的场景。
- 网络基础需要把 MAC、IP、ARP、ICMP、子网掩码、路由和 NAT 串起来理解。
- 子网计算的关键是 IP 地址与子网掩码做按位与，得到网络号。

## 关系表

| 条目     | 上游依赖              | 下游用途              |
| ------ | ----------------- | ----------------- |
| RPM    | 已编译软件包、RPM 数据库    | 安装、卸载、查询本地软件包     |
| YUM    | RPM、Repository、缓存 | 自动解决依赖、批量安装升级     |
| 源码安装   | 编译环境、Makefile     | 自定义安装路径、启用特定模块    |
| MAC 地址 | 网络接口层             | 局域网通信、ARP 解析      |
| IP 地址  | 网络层               | 跨网段通信、路由选择        |
| 子网掩码   | IP 地址             | 计算网络号、广播地址、可用主机范围 |
| 路由     | 网段、网关             | 决定数据包转发方向         |
| NAT    | 私有地址、公网地址         | 内网访问公网、公网映射内网服务   |

## RPM 软件包管理

### RPM 概念

RPM 全称 RedHat Package Manager。开发者在指定系统环境中提前编译好软件，并把软件文件、依赖信息、版本信息等打包成 `.rpm` 文件。

RPM 安装时会检查依赖和系统环境，安装完成后把软件信息记录到 RPM 数据库中，便于查询和卸载。

RPM 特点：

| 优点         | 缺点                 |
| ---------- | ------------------ |
| 安装、卸载、查询方便 | 不同系统版本、平台之间包不通用    |
| 有数据库记录软件文件 | 不能自动解决依赖           |
| 适合标准化系统软件  | 安装路径通常固定，不适合随意指定目录 |

RPM 数据库：

```bash
/var/lib/rpm
```

> **注意**\
> RPM 可以通过 `--prefix` 指定路径，但不推荐。多数 RPM 包按系统规范打包，强行改路径容易引发配置、依赖和服务管理问题。

### RPM 包命名

示例：

```text
dhcp-server-4.3.6-30.el8.x86_64.rpm
```

| 部分            | 含义       |
| ------------- | -------- |
| `dhcp-server` | 软件名      |
| `4.3.6`       | 软件版本     |
| `30`          | 编译次数/发布号 |
| `el8`         | 适用系统版本   |
| `x86_64`      | 适用平台架构   |
| `rpm`         | 后缀名      |

### RPM 文件安装位置

| 类型      | 常见目录                              |
| ------- | --------------------------------- |
| 配置文件    | `/etc`                            |
| 执行文件    | `/usr/bin`                        |
| 链接库文件   | `/usr/lib`                        |
| RPM 数据库 | `/var/lib/rpm`                    |
| 帮助文档    | `/usr/share/man`、`/usr/share/doc` |

### RPM 命令

安装：

```bash
rpm -ivh dhcp-server-4.3.6-30.el8.x86_64.rpm
```

卸载：

```bash
rpm -e dhcp-server
rpm -e --nodeps dhcp-server
```

常用选项：

| 选项               | 含义             |
| ---------------- | -------------- |
| `-i`             | 安装             |
| `-v`             | 显示详细信息         |
| `-h`             | 显示安装进度         |
| `-e`             | 卸载             |
| `-U`             | 升级；没有安装则安装     |
| `-F`             | 只在已安装旧版本时升级    |
| `--prefix`       | 指定安装路径，不建议     |
| `--nodeps`       | 忽略依赖关系         |
| `--replacefiles` | 覆盖文件           |
| `--replacepkgs`  | 重装/修复包         |
| `--force`        | 强制             |
| `--test`         | 测试，不实际安装       |
| `-q`             | 查询是否安装         |
| `-qi`            | 查看软件包信息        |
| `-ql`            | 查看软件包包含的文件列表   |
| `-qc`            | 查看软件包配置文件      |
| `-qa`            | 查看本机所有已安装包     |
| `-qf`            | 查询某个文件属于哪个已安装包 |

NOKEY 警告：

```text
Header V3 RSA/SHA256 Signature ... NOKEY
```

表示系统没有对应 GPG 公钥，属于签名验证提示，不等于安装必然失败。

## YUM 软件包管理

### YUM 概念

YUM 全称 Yellowdog Updater Modified。它基于 RPM，但比 RPM 更适合日常软件管理，因为它可以通过仓库自动处理依赖。

YUM 与 RPM 的关系：

```text
YUM -> 调用 RPM 安装包 + 自动查找依赖 + 管理仓库与缓存
```

YUM 安装的软件最终仍按 RPM 包默认路径安装，不能随意指定目录。

### YUM 组成

| 组件             | 说明                      |
| -------------- | ----------------------- |
| YUM Repository | 软件仓库，可以是本地目录、HTTP、FTP 等 |
| yum 命令         | 用户操作入口                  |
| YUM 插件         | 扩展功能，例如选择最快源            |
| YUM cache      | 缓存软件包元数据和下载数据           |

缓存目录：

```bash
/var/cache/yum
```

### 配置网络源

备份旧源：

```bash
mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
```

下载阿里云 CentOS 7 源：

```bash
wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo
```

生成缓存：

```bash
yum makecache
```

也可以使用 curl：

```bash
curl -s -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
```

EPEL 源慢时：

```bash
curl -s -o /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-7.repo
```

添加或更新源后：

```bash
yum clean all && yum makecache
```

### 清理缓存

```bash
yum clean all
yum makecache
```

`yum clean all` 会清除 yum 缓存和下载的软件包。清完后不生成缓存，后续安装软件可能变慢。

### 安装、卸载、查询、更新

```bash
yum install epel-release -y
yum remove epel-release -y
yum list | grep xxx
yum info epel-release
yum search epel-release
yum update 软件包名称 -y
yum update -y
yum repolist
```

| 命令                 | 作用          |
| ------------------ | ----------- |
| `yum install 包 -y` | 安装          |
| `yum remove 包 -y`  | 卸载          |
| `yum list`         | 查看仓库和已安装包列表 |
| `yum info 包`       | 查看包信息       |
| `yum search 关键词`   | 搜索软件包       |
| `yum update 包 -y`  | 更新指定包       |
| `yum update -y`    | 更新所有包       |
| `yum repolist`     | 查看可用仓库      |

### EPEL 源

EPEL 是 CentOS/RHEL 常用扩展源：

```bash
yum install epel-release -y
```

### 软件组管理

```bash
yum groupinstall 软件集合名称 -y
yum groupremove 软件集合名称 -y
yum groupupdate 软件集合名称 -y
yum groupinfo 软件集合名称
yum grouplist
```

## 源码安装软件

### 使用场景

通常优先使用 YUM。源码安装适合以下情况：

- yum 仓库里的版本过低。
- yum 包默认未启用需要的功能参数。
- 需要自定义安装目录、运行用户、模块或编译选项。

源码安装也会遇到依赖问题；如果缺少编译依赖或运行依赖，编译或运行会失败。

### 基本流程

```bash
# 下载源码包并解压
cd 源码目录
./configure --prefix=安装目录 --user=运行用户 --group=运行用户组
make
make install
make clean
```

| 步骤             | 作用               |
| -------------- | ---------------- |
| `./configure`  | 检测环境，生成 Makefile |
| `make`         | 根据 Makefile 编译源码 |
| `make install` | 安装编译结果           |
| `make clean`   | 清理编译过程产生的临时文件    |

常见 `./configure` 参数：

| 参数            | 含义      |
| ------------- | ------- |
| `--prefix=路径` | 指定安装目录  |
| `--user=用户`   | 指定运行用户  |
| `--group=用户组` | 指定运行用户组 |

卸载源码安装的软件通常直接删除安装目录，具体还要看软件是否提供 `make uninstall` 或额外卸载脚本。

## 网络接口层

| 项      | 说明                |
| ------ | ----------------- |
| 地址     | MAC 地址            |
| 基本数据单位 | 帧                 |
| 主要协议   | 以太网协议             |
| 主要设备   | 交换机               |
| MTU    | 以太网常见最大传输单元为 1500 |

MAC 地址：

- 每个网卡设备通常有唯一 MAC 地址。
- 长度 48 位。
- 通常用十六进制表示。

## 网络层

网络层目标是实现两个端系统之间的数据传送，核心功能包括寻址、路由选择、连接相关处理等。

| 协议   | 作用                       |
| ---- | ------------------------ |
| IP   | 网络层寻址与转发                 |
| ARP  | IP 地址解析为 MAC 地址          |
| RARP | 逆地址解析                    |
| ICMP | 网络控制与差错报告，`ping` 使用 ICMP |

主要设备：

```text
路由器
```

## IP 地址分类

IPv4 范围：

```text
0.0.0.0 - 255.255.255.255
```

特殊地址：

| 地址                | 含义     |
| ----------------- | ------ |
| `255.255.255.255` | 受限广播地址 |
| `127.0.0.1`       | 本机环回地址 |

传统分类：

| 类别 | 二进制前缀  | 地址范围                            | 用途   |
| -- | ------ | ------------------------------- | ---- |
| A  | `0`    | `0.0.0.0` - `127.255.255.255`   | 大型网络 |
| B  | `10`   | `128.0.0.0` - `191.255.255.255` | 中型网络 |
| C  | `110`  | `192.0.0.0` - `223.255.255.255` | 小型网络 |
| D  | `1110` | `224.0.0.0` - `239.255.255.255` | 组播   |
| E  | `1111` | `240.0.0.0` - `255.255.255.255` | 保留   |

## 私有地址

私有地址不会直接在公网路由中使用，通常只在内网使用。

| 类别      | CIDR             | 范围                                |
| ------- | ---------------- | --------------------------------- |
| A 类私有地址 | `10.0.0.0/8`     | `10.0.0.0` - `10.255.255.255`     |
| B 类私有地址 | `172.16.0.0/12`  | `172.16.0.0` - `172.31.255.255`   |
| C 类私有地址 | `192.168.0.0/16` | `192.168.0.0` - `192.168.255.255` |

## 子网掩码与网络号

`/24` 表示子网掩码：

```text
255.255.255.0
```

例：

```yaml
IP:   192.168.31.22
Mask: 255.255.255.0
网络号: 192.168.31.0
广播地址: 192.168.31.255
主机范围: 192.168.31.1 - 192.168.31.254
```

计算方式：

```text
IP 地址二进制
AND 子网掩码二进制
= 网络号
```

## 子网计算示例

### 192.168.31.46 / 255.255.255.248

```yaml
IP:   192.168.31.46
Mask: 255.255.255.248
```

最后一段块大小：

```text
256 - 248 = 8
```

网段边界：

```text
...32, 40, 48, 56...
```

46 落在 `40-47` 中：

| 项    | 结果                              |
| ---- | ------------------------------- |
| 网络号  | `192.168.31.40`                 |
| 可用主机 | `192.168.31.41 - 192.168.31.46` |
| 广播地址 | `192.168.31.47`                 |

### 192.168.21.169 / 255.255.255.224

块大小：

```text
256 - 224 = 32
```

169 落在 `160-191` 中：

| 项    | 结果                                |
| ---- | --------------------------------- |
| 网络号  | `192.168.21.160`                  |
| 可用主机 | `192.168.21.161 - 192.168.21.190` |
| 广播地址 | `192.168.21.191`                  |

### 10.100.111.245 / 255.255.192.0

第三段块大小：

```text
256 - 192 = 64
```

111 落在 `64-127` 中：

| 项    | 结果                             |
| ---- | ------------------------------ |
| 网络号  | `10.100.64.0`                  |
| 可用主机 | `10.100.64.1 - 10.100.127.254` |
| 广播地址 | `10.100.127.255`               |

## 路由表

查看路由表：

```bash
route -n
```

`-n` 表示不做域名解析，直接显示数字 IP。

## 临时静态路由

```bash
route add -net 22.22.22.0 netmask 255.255.255.0 gw 192.168.8.1
```

含义：去往 `22.22.22.0/24` 网段的数据，下一跳交给网关 `192.168.8.1`。

## 永久静态路由

CentOS 7 永久静态路由写入：

```bash
/etc/sysconfig/network-scripts/route-网卡名
```

示例：网卡名 `eth0`。

```bash
vim /etc/sysconfig/network-scripts/route-eth0
```

```text
10.15.150.0/24 via 192.168.8.1 dev eth0
10.25.250.0/24 via 192.168.8.1 dev eth0
```

重启网络服务：

```bash
systemctl restart network
```

## 虚拟 IP

临时添加虚拟 IP：

```bash
ifconfig ens33:1 192.168.8.5 netmask 255.255.255.0 up
```

这种方式重启后失效。长期配置应写入网卡配置文件或使用 NetworkManager/nmcli 等方式管理。

## NAT

NAT 用于地址转换，常见作用：

- 内网私有地址访问公网。
- 多台内网主机共享一个公网出口。
- 端口映射，把公网访问转发到内网服务。

典型关系：

```text
内网私有 IP -> NAT 设备转换 -> 公网 IP
```

## 易错点

| 场景                            | 说明                         |
| ----------------------------- | -------------------------- |
| rpm 忽略依赖安装                    | 包可能装上但无法运行                 |
| 误用 `--nodeps`                 | 只适合明确知道依赖风险时使用             |
| yum 源更新后不 makecache           | 安装可能慢或读旧缓存                 |
| yum update -y 随便全量升级          | 生产环境可能引入版本变化风险             |
| 源码安装后不知道文件在哪                  | 编译前明确 `--prefix`           |
| 子网掩码 `/25` 写成 `255.255.255.0` | `/25` 应是 `255.255.255.128` |
| 网络号把主机位算进去                    | IP 与子网掩码按位与后才是网络号          |
| 临时路由/虚拟 IP 重启丢失               | 永久配置要写配置文件                 |
