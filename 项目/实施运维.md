# 实施运维

# 实施交付过程

**了解项目基本情况**

- 硬件情况，是否需要硬件部署，硬件配置如何
- 软件情况，版本，接口对接
- 验收标准
- 工期安排，交付日期

**了解客户基本情况**

- 客户对项目态度是否主动
- 客户的对接人和负责人
- 客户的部门配合程度
- 客户对项目的准备程度，资源、权限等

**系统环境确认**

- 硬件资源：如果客户自行提供，给客户一份环境配置清单；如果由公司提供，设备发货到现场前做好配置
- 项目开始前再次确认相关信息，包括服务器配置，软件版本，网络通畅等

**在测试环境模拟部署**

- 安装Centos7
- 精简开机启动项，关闭不需要的系统服务
- 优化内核参数，进行安全加固
- 开启防火墙并配置
- 安装对应软件

**项目正式开始部署**

- 与对接人、负责人沟通，制定项目部署计划表，明确部署时间、步骤、责任人、注意事项等
- 严格按照部署计划表启动项目部署工作，有序完成生产环境全量部署、调试、功能验证等步骤

**培训交付，保证客户能够独立使用系统**

- 确定培训时间、地点、人员，提前通知客户，下发培训安排
- 准备培训材料，包括系统操作手册、培训ppt、会议签到表、答疑材料
- 搭建培训环境，准备测试演示数据，完整演示系统全部功能
- 完成功能讲解、实操教学、答疑，确保客户熟练操作

**项目验收**

- 提前准备全套材料并打印，告知客户盖章要求
- 与客户确认试运行周期，在周期内跟进系统运行状态和数据，出具《系统试运行报告》
- 试运行完成，与客户对接项目的整体工作。客户确认系统运行正常、符合验收标准后签署项目验收报告，项目正式交付完成

进入维保时期后需要编写巡检报告

# 运维工作内容

项目验收交付完成后就进入长期运维阶段

核心目标：**系统稳定运行不宕机、数据不丢失、业务正常使用、问题及时处理**

1. **系统巡检**。包括服务器性能、服务器各种数据状态、系统软件平台（数据库、中间件）、对业务各个模块、使用情况进行巡检定期输出巡检报告交给客户
2. **业务升级**。配合研发对公司业务各种变更/升级/测试，公司各种业务板块针对用户的反馈不定期会有升级变更，配合研发同事做调试、做测试
3. **脚本编写**。据客户需要，进行数据库的增删改查等操作，编写SQL脚本，进行数据备份。根据用户需求，进行shell脚本的编写
4. **应急响应**。各种故障的排查、处理。处理用户反馈的各项问题，包括用户在使用产品方面的问题、遇到的BUG、需要的各种技术支持等等
5. **安装部署**。服务器业务软件安装部署及调试，产品的安装部署，有需要到用户现场的，需要到现场进行安装实施、培训、验收等相关工作
6. **设备更换**。需要设备（服务器，网络设备，安全产品）上架下架/更换的，需要配合公司产品部门做好相关工作
7. **状态监控**。部署 zabbix 监控服务，负责监控公司的各类服务系统运行状态并进行维护
8. **测试环境**。linux系统安装等等，业务上线之前在测试环境下进行测试

# 巡检内容

基本系统指标，cpu、内存、磁盘、服务运行状态、服务运行时间等，自动备份脚本、计划任务等，针对不同服务器有不同的额外检查内容

1. web服务器：流量、访问量
2. 数据库服务器：数据库运行状态，数据库性能
3. redis服务器：同数据库服务器
4. zabbix服务器：LNMP平台运行状态
